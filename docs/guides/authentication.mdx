ShiftCut uses a Verblike credential for premium voiceover (TTS) and the music / sound-effects library. Other providers are optional, and **everything runs without any key** — voice and music fall back to fully local engines. This page covers signing in, the keys each capability uses, and the order they resolve.

## Sign in

Signing in is the same OAuth step as creating an account — new users land on the sign-up screen.

<Steps>
  <Step title="Sign in">
    The default flow opens your browser for OAuth and captures the token on a loopback port:

    ```bash
    npx shiftcut auth login
    # ✓ Signed in.
    ```

    For CI or headless machines, save a long-lived API key instead:

    ```bash
    npx shiftcut auth login --api-key            # hidden-input prompt
    echo "$VERBLIKE_API_KEY" | npx shiftcut auth login --api-key   # from stdin
    ```
  </Step>
  <Step title="Confirm what's configured">
    ```bash
    npx shiftcut auth status
    ```

    Shows the active credential's source and verified identity, and — when you're signed out — which local engines voice and music will use. Add `--json` for `{ configured, recommended_action, offline_engines }` in scripts.
  </Step>
</Steps>

The credential lives in `~/.verblike/credentials` (mode `0600`) — no per-repo `.env` to manage. Browser OAuth is a `shiftcut auth login` feature. The separate [`verblike` CLI](https://github.com/Verblike/verblike-cli) (its own install — there's no `npx verblike`) is API-key-only, so `verblike auth login` just stores a key you paste. Both read the same `~/.verblike/credentials`, so signing in with one carries to the other.

<Tip>
  No account needed to try ShiftCut. With no credential, voice uses **Kokoro** and music uses **MusicGen**, both fully local and offline — see [Working offline](#working-offline).
</Tip>

## How credentials resolve

The Verblike credential drives TTS and music / SFX **retrieval**. It resolves first-match-wins:

1. `VERBLIKE_API_KEY` — environment variable
2. `SHIFTCUT_API_KEY` — alias, for parity with other tools
3. `~/.verblike/credentials` — written by `shiftcut auth login` (or `verblike auth login`)

Point at a different config directory with `VERBLIKE_CONFIG_DIR`, or a different backend with `VERBLIKE_API_URL`.

## Keys by capability

Each capability picks the **first available provider** in order; the last is always a local engine that needs no key. Cloud providers below the Verblike line need their own key *and* a local Python dependency.

| Capability | Provider order | Key(s) — first match wins | Local dependency |
|------------|----------------|---------------------------|------------------|
| **Voice (TTS)** | Verblike → ElevenLabs → Kokoro | `VERBLIKE_API_KEY` → `SHIFTCUT_API_KEY` → `~/.verblike` · then `ELEVENLABS_API_KEY` | Kokoro: `pip install kokoro-onnx soundfile` |
| **Music (BGM)** | Verblike library → Lyria → MusicGen | Verblike credential (above) · then `GEMINI_API_KEY` → `GOOGLE_API_KEY` | MusicGen: `pip install transformers torch soundfile numpy` |
| **Sound effects** | Verblike library → bundled library | Verblike credential (above) | bundled — no deps |
| **Capture descriptions** | OpenRouter → Gemini | `OPENROUTER_API_KEY` → `GEMINI_API_KEY` | — (optional; for [website capture](/guides/website-to-video)) |

Run `npx shiftcut doctor` to check which local dependencies are installed. The media skills also run `shiftcut auth status` as a preflight before generating, so you always know whether a run will use Verblike or a local engine before it starts.

## Working offline

No key configured is a normal state, not an error. The workflow runs entirely on local models:

- **Voice** — Kokoro-82M (54 voices), with Whisper for word-level caption alignment.
- **Music** — MusicGen (`facebook/musicgen-small`).
- **Sound effects** — a bundled library.

Local engines are free and offline; Verblike gives higher-quality voices and a professionally produced music library. Sign in any time to switch a project from local to Verblike.

## Environment variables

| Variable | Used for |
|----------|----------|
| `VERBLIKE_API_KEY` | Verblike credential — voice + music/SFX retrieval. Highest priority. |
| `SHIFTCUT_API_KEY` | Alias for `VERBLIKE_API_KEY`. |
| `VERBLIKE_API_URL` | API base URL (default `https://shiftcut.verblike.com`). |
| `VERBLIKE_CONFIG_DIR` | Credentials directory (default `~/.verblike`). |
| `ELEVENLABS_API_KEY` | ElevenLabs TTS, used when no Verblike credential is present. |
| `GEMINI_API_KEY` / `GOOGLE_API_KEY` | Lyria music generation (and capture descriptions). |
| `OPENROUTER_API_KEY` | Capture descriptions; takes priority over Gemini for that step. |

See the [`shiftcut auth`](/packages/cli#shiftcut-auth) command reference for subcommand details, and [Cloud rendering](/deploy/cloud) for using the same credential to render in Verblike's cloud.
